NodeValt

Reduza a repeticao de node_modules entre projetos com store global, materializacao segura, backups e um observer discreto no macOS.

Instalar agora Ver codigo
npm lockfile v2/v3 hardlinks e backup daemon para macOS

Problema

Cada projeto guarda o mesmo peso de novo.

NodeValt troca o desperdicio por um fluxo local controlado: escaneia projetos, popula um store global, materializa dependencias sob demanda e valida o estado com doctor.

Fluxo

O ciclo que mantem tudo reversivel.

01

Escaneia

Mapeia projetos Node.js e lockfiles sem assumir controle de repos que nao usam Node.

02

Popula o store

Baixa pacotes resolvidos no lockfile e guarda conteudo em um diretorio global local.

03

Materializa seguro

Cria a arvore gerenciada com hardlinks ou copia e preserva backup do estado anterior.

04

Observa

O daemon acompanha mudancas relevantes e repete o scan em intervalos configuraveis.

Instalacao

Comece pelo global install.

Instale do GitHub, inicialize o store e deixe o observer rodando quando fizer sentido. 

npm install -g https://github.com/victorazesc/nodevalt/archive/refs/heads/main.tar.gz
nodevalt init
nodevalt daemon install
nodevalt daemon status

Comandos

Superficie pequena, util no dia a dia.

nodevalt scan ~/projetosdescobre projetos candidatos
nodevalt store populate ./apppreenche o store global
nodevalt materialize ./appativa a arvore gerenciada
nodevalt restore ./appvolta para o backup local
nodevalt doctor ./appvalida links, store e estado
nodevalt gcremove pacotes nao usados

FAQ

Promessa simples.

Ele substitui npm ou yarn?

Nao. Ele trabalha em volta do lockfile e do node_modules local.

Da para desfazer?

Sim. A materializacao preserva backup e o comando restore volta o projeto.

Onde ficam os dados?

Por padrao em ~/.nodevalt-global-shell, com packages, projetos e logs.
